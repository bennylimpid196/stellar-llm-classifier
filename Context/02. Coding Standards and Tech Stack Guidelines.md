## 1. Core Philosophy and Language

- **Language Requirement:** ALL code, variables, functions, inline comments, docstrings, and JSON outputs MUST be written entirely in **English**.
    
- **Tone and Style:** Maintain a strictly professional, academic, and software-engineering-oriented tone. Do not use emojis in the code, comments, or terminal outputs.
    
- **Modularity (Single Responsibility Principle):** Code must be highly modular and object-oriented. Each mathematical process or Agent (e.g., Continuum Normalization, Voigt Fitting) must be encapsulated in its own Class.
    
- **Commenting Rule:** Comment ONLY mathematical operations, physical assumptions, or domain-specific logic that aren't self-evident from variable names. Do not over-comment obvious Python syntax.
    

## 2. Tech Stack Hierarchy and Preferences

The project relies on standard astronomical and data science libraries in Python. Follow this strict hierarchy for tool selection:

1. **Spectral Fitting (Lines):** PREFER `astropy.modeling` (e.g., `models.Voigt1D`, `fitting.LevMarLSQFitter`) over `scipy.optimize`. Astropy handles units and astronomical parameter bounds natively.
    
2. **Continuum Fitting:** PREFER `scipy.interpolate` (e.g., `CubicSpline`, `splrep`) or `numpy.polynomial.chebyshev` combined with custom sigma-clipping over Astropy models for speed.
    
3. **Data Manipulation:** Use `pandas` for catalog-level tabular data and `numpy` for array-level spectral arrays.
    

## 3. Anti-Patterns (STRICTLY FORBIDDEN)

- **NO `plt.show()` in production classes:** Visualization blocks automated pipelines. If plotting is needed for debugging, it must be an optional method that saves to disk (e.g., `fig.savefig()`), never a blocking UI call.
    
- **NO Hardcoded Paths:** Do not hardcode `/users/data/gaia.csv`. Always pass file paths as arguments or use the `pathlib` module.
    
- **NO Silent Failures:** Never use naked `except:` or `except Exception: pass`. Catch specific exceptions (e.g., `OptimizeWarning`, `ValueError`) and return a structured failed state or `NaN`.
    
- **NO Print-based Logging:** Do not use `print()` for process tracking in production classes. Use Python's built-in `logging` module or return status messages in the output dictionary.

## 5. Code Benchmark / Class Structure Template

Whenever you generate a class for an Agent, follow this exact structural, typing, and documentation standard:

Python

```
import numpy as np
import logging
from typing import Dict, Any, Optional
from astropy.modeling import models, fitting

logger = logging.getLogger(__name__)

class SpectralLineFitter:
    """
    Fits a physical profile (Voigt) to a specific spectral line array.
    
    Attributes:
        rest_wavelength (float): The laboratory wavelength of the line in nm.
    """
    
    def __init__(self, rest_wavelength: float):
        self.rest_wavelength = rest_wavelength

    def fit_line(self, wavelength: np.ndarray, flux: np.ndarray) -> Dict[str, Any]:
        """
        Executes the Levenberg-Marquardt fit on the provided spectral array.

        Args:
            wavelength (np.ndarray): Array of wavelengths in nm.
            flux (np.ndarray): Array of normalized flux values.

        Returns:
            Dict[str, Any]: A JSON-serializable dictionary containing the fit 
                            parameters, equivalent width, and success flags.
        """
        # Initialize default failure state
        result = {
            "module": "SpectralLineFitter",
            "status": "failed",
            "measurements": {
                "equivalent_width_nm": np.nan,
                "fwhm_nm": np.nan
            },
            "flags": {"fit_converged": False}
        }

        if len(wavelength) != len(flux) or len(wavelength) < 5:
            logger.warning("Invalid input arrays. Returning failed state.")
            return result

        try:
            # Initialize Astropy Voigt model and fitter
            model_init = models.Voigt1D(x_0=self.rest_wavelength, amplitude_L=-0.5)
            fitter = fitting.LevMarLSQFitter()
            
            # Perform fit
            fitted_model = fitter(model_init, wavelength, flux)
            
            # Check for fitter warnings/failures
            if fitter.fit_info['ierr'] not in [1, 2, 3, 4]:
                logger.warning("LevMarLSQFitter failed to converge.")
                return
```

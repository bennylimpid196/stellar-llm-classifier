## 1. Overview

This document defines the exact schema, data types, and physical units of the input data extracted from the Gaia DR3 archive. It serves as the single source of truth for the input parameters expected by the Hard Computing (HC) modules (`AstrometryAgent`, `ContinuumAgent`, `LineAgent`, and `BinaryDetectorAgent`), as well as the parameters passed directly to the Soft Computing (SC) layer.

The data is logically split into two structures: **Tabular Metadata** (scalar values per star) and **Spectral Arrays** (1D arrays of flux and wavelength).

## 2. Tabular Metadata Dictionary (Pandas DataFrame / CSV)

|**Column Name**|**Data Type**|**Units**|**Description & Edge Cases**|
|---|---|---|---|
|`source_id`|`int64`|None|Unique Gaia DR3 identifier. Do not cast to float.|
|`ra`|`float`|degrees|Right Ascension (0 to 360).|
|`dec`|`float`|degrees|Declination (-90 to +90).|
|`parallax`|`float`|mas|Parallax. **WARNING:** Can be $\le 0$ for distant/noisy sources.|
|`parallax_error`|`float`|mas|Standard error of parallax.|
|`pmra`|`float`|mas/yr|Proper motion in RA. Already corrected by $\cos(\delta)$ in DR3 ($\mu_{\alpha*}$), do not re-apply cosine.|
|`pmdec`|`float`|mas/yr|Proper motion in Dec ($\mu_\delta$).|
|`ruwe`|`float`|None|Renormalised Unit Weight Error. Values $>1.4$ imply bad fit/binarity.|
|`phot_g_mean_mag`|`float`|mag|Apparent G-band magnitude. Lower is brighter.|
|`ebpminrp_gspphot`|`float`|mag|Extinction $E(BP-RP)$. **WARNING:** Highly prone to `NaN`.|
|`teff_gspphot`|`float`|K|Effective temperature. Prone to `NaN`.|
|`logg_gspphot`|`float`|dex|Surface gravity $\log(g)$ in cgs. _Bypasses HC logic; passed directly to SC for luminosity classification._|
|`mh_gspphot`|`float`|dex|Global metallicity $[M/H]$.|
|`alphafe_gspspec`|`float`|dex|Alpha-element enhancement $[\alpha/Fe]$.|
|`fem_gspspec`|`float`|dex|Iron abundance $[Fe/M]$ from spectroscopy. _Bypasses HC logic; passed directly to SC for cross-validation with mh_gspphot._|
|`ew_espels_halpha`|`float`|Å|ESP-ELS H-alpha Equivalent Width. **WARNING:** DR3 convention specifies positive values = absorption, negative values = real emission.|
|`ew_espels_halpha_flag`|`int`|None|Quality flag for H-alpha. **WARNING:** Use only if `flag == 0`.|
|`radial_velocity_error`|`float`|km/s|Error in radial velocity. Can be `NaN` if no RVS data exists.|
|`rv_nb_transits`|`float`|count|Number of RV transits. Arrives as float (e.g., `5.0`), cast to `int` defensively.|
|`nss_solution_type`|`string`|None|Non-Single Star flag. **WARNING:** Can be empty string `""`, `"None"`, or `NaN`.|

## 3. Spectral Array Specifications

For the `ContinuumAgent` and `LineAgent`, the pipeline expects 1D NumPy arrays extracted from the Gaia XP (BP/RP) or RVS continuous spectra datasets.

- `wavelength`: `np.ndarray` (float). Wavelengths must be uniformly converted to **nanometers (nm)** prior to injection.
    
- `flux`: `np.ndarray` (float). Absolute flux. Will be normalized to 1.0 by the `ContinuumAgent`.

**Sample Analysis for the LLM:**

1. **Row 1 (Source `...0528`):** A clean, ideal main-sequence star. High SNR parallax, good RUWE, valid H-alpha measurement (absorption), and no NSS solution (empty string).
    
2. **Row 2 (Source `...9999`):** A highly problematic target. **Negative parallax**, `NaN` in extinction, `NaN` in all chemical and spectroscopic fields, and a high RUWE (2.80). Modules must use strict guards (e.g., `if parallax <= 0`) and fallback to `NaN`/`None` safely. _(Note: trailing commas represent the `NaN` values)._
    
3. **Row 3 (Source `...4444`):** A confirmed binary system. Valid parameters, but high `radial_velocity_error` despite many transits (`25.0`), and an explicit `Orbital` string in `nss_solution_type`.

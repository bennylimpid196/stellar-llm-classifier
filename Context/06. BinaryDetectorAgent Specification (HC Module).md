## 1. Overview

El `BinaryDetectorAgent` es un módulo de Hard Computing (HC) diseñado para identificar sistemas binarios o múltiples no resueltos. Previene que la capa de Soft Computing (SC) clasifique erróneamente un espectro compuesto (una "quimera") como una estrella física única. El agente evalúa indicadores de calidad astrométrica, variabilidad en la velocidad radial y soluciones oficiales del catálogo Non-Single Star (NSS).

## 2. Metodología Matemática y Heurística

### 2.1 Tambaleo Astrométrico (RUWE)

El _Renormalised Unit Weight Error_ (RUWE) mide la calidad del ajuste astrométrico para un modelo de estrella única. Sistemas múltiples no resueltos causan que el fotocentro "tambalee".

- **Umbral:** Según Lindegren et al. (2021), un `ruwe > 1.4` es un fuerte indicador de una fuente no simple o problemática.
    

### 2.2 Variabilidad Espectroscópica Adaptativa (Velocidad Radial)

Si una estrella es binaria, su Velocidad Radial (RV) cambia con el tiempo debido al movimiento orbital. Gaia DR3 reporta el error en la RV media (`radial_velocity_error`) y el número de tránsitos (`rv_nb_transits`).

- **Umbral Adaptativo basado en Teff:** El error natural de la RV depende fuertemente del tipo espectral (disponibilidad y nitidez de las líneas).
    
    - Estrellas frías ($T_{eff} < 4000$ K): Umbral estricto $\sim 2.0$ km/s.
        
    - Estrellas estándar ($4000 \le T_{eff} < 7000$ K): Umbral base $\sim 5.0$ km/s.
        
    - Estrellas calientes ($T_{eff} \ge 7000$ K): Umbral permisivo $\sim 10.0$ km/s (evita falsos positivos debidos a escasez de líneas).
        

### 2.3 Soluciones Oficiales NSS

Si la columna `nss_solution_type` contiene una cadena de texto válida (ej. 'Orbital', 'AstroSpectroSB1'), la colaboración Gaia ya ha confirmado matemáticamente la naturaleza binaria de la fuente.

## 3. Banderas Lógicas (Predigestión)

- **`is_astrometric_binary`**: `True` si `ruwe > 1.4`.
    
- **`is_rv_variable`**: `True` si los tránsitos $\ge 5$ Y el error de RV supera el umbral adaptativo dictado por la $T_{eff}$.
    
- **`is_confirmed_nss`**: `True` si existe una solución válida en el catálogo NSS.
    
- **`is_binary_candidate`**: La bandera maestra. `True` si _cualquiera_ de los indicadores anteriores es verdadero.
    

## 4. Plantilla de Implementación (Python)

```python
import numpy as np
import logging
import pandas as pd
from typing import Dict, Any, Optional

logger = logging.getLogger(__name__)

class BinaryDetectorAgent:
    """
    HC Module to detect unresolved multiple systems using Gaia DR3 catalog indicators.
    Features an adaptive RV error threshold based on effective temperature.
    """

    def __init__(self, ruwe_threshold: float = 1.4, min_rv_transits: int = 5):
        self.ruwe_threshold = ruwe_threshold
        self.min_rv_transits = min_rv_transits
        self.default_rv_error_threshold = 5.0

    def _check_nss_validity(self, nss_type: Any) -> bool:
        """Safely checks if the NSS solution type is a valid populated string."""
        if pd.isna(nss_type) or nss_type is None:
            return False
        nss_str = str(nss_type).strip().lower()
        if nss_str in ['', 'none', 'nan', 'null']:
            return False
        return True

    def _get_adaptive_rv_threshold(self, teff: float) -> float: 
        """ 
        Returns Teff-dependent RV error threshold. Caveats: 
        - Hot stars (>=7000 K): threshold raised to 10.0 km/s to avoid false positives from sparse spectral lines. 
        - Cool stars (<4000 K): threshold lowered to 2.0 km/s, but chromospheric activity in young/active M-type stars can produce intrinsic RV jitter of 1-3 km/s even for single stars. is_rv_variable alone is insufficient for this population; always cross-check with is_astrometric_binary. 
        """
        if pd.isna(teff):
            return self.default_rv_error_threshold
        if teff >= 7000:
            return 10.0  # Hot stars: few broad lines, high natural error
        elif teff < 4000:
            return 2.0   # Cool stars: many sharp lines, low natural error
        return self.default_rv_error_threshold

    def process(self, data: Dict[str, Any], source_id: str = "UNKNOWN") -> Dict[str, Any]:
        """Evaluates binarity criteria and populates the master boolean flags."""
        res = {
            "module": "BinaryDetectorAgent",
            "status": "failed",
            "measurements": {
                "ruwe": None,
                "rv_error_km_s": None,
                "rv_nb_transits": None,
                "nss_solution": None
            },
            "logical_flags": {
                "is_astrometric_binary": False,
                "is_rv_variable": False,
                "is_confirmed_nss": False,
                "is_binary_candidate": False
            }
        }

        try: 
            # 1. Extract Data (using .get() to handle missing columns gracefully) 
            ruwe = data.get('ruwe', np.nan) 
            rv_err = data.get('radial_velocity_error', np.nan) 
            rv_transits = data.get('rv_nb_transits', 0) 
            nss_type = data.get('nss_solution_type', None) 
            
            # 2. Astrometric Evaluation 
            is_astro_binary = False 
            if not pd.isna(ruwe) and ruwe > self.ruwe_threshold: 
                is_astro_binary = True 
                
            # 3. Spectroscopic Evaluation 
            is_rv_var = False 
            if not pd.isna(rv_err) and not pd.isna(rv_transits): 
                if rv_transits >= self.min_rv_transits and rv_err > self.rv_error_threshold: 
                    is_rv_var = True 
                    
            # 4. Official NSS Evaluation 
            is_nss = self._check_nss_validity(nss_type) 
            
            # 5. Master Flag 
            master_binary_flag = is_astro_binary or is_rv_var or is_nss 
            
            # 6. Populate Contract 
            res["measurements"] = { 
                "ruwe": float(ruwe) if not pd.isna(ruwe) else np.nan, 
                "rv_error_km_s": float(rv_err) if not pd.isna(rv_err) else np.nan, 
                "nss_solution": str(nss_type) if is_nss else "None" 
            } 
            
            res["logical_flags"] = { 
                "is_astrometric_binary": is_astro_binary, 
                "is_rv_variable": is_rv_var, 
                "is_confirmed_nss": is_nss, 
                "is_binary_candidate": master_binary_flag 
            } 
            
            res["status"] = "success" 
            
        except Exception as e: 
            logger.error(f"[{source_id}] BinaryDetectorAgent critical failure: {e}") 
            
        return res
```
